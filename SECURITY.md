# Security Policy

## Supported Versions

The following table indicates which versions of ID-oto Automation currently receive security updates.

| Version | Supported |
|---------|-----------|
| Development (main) | ✅ |

> Only the latest stable release is supported with security updates. Users are encouraged to upgrade to the newest version whenever possible.

---

## Reporting a Vulnerability

The ID-oto Automation development team values responsible security disclosure. If you discover a security vulnerability, please report it privately.

### How to Report

Please **do not report security vulnerabilities through public GitHub Issues**.

Instead, report the issue by:

- Opening a **GitHub Private Security Advisory** (if available), or
- Contacting the project maintainers through the project's official communication channel.

Please include the following information in your report:

- Description of the vulnerability
- Steps to reproduce the issue
- Expected and actual behavior
- Affected version(s)
- Screenshots, logs, or proof-of-concept (if applicable)
- Any suggested mitigation (optional)

---

## Response Timeline

The development team aims to:

- Acknowledge reports within **3–5 business days**.
- Investigate and verify the reported vulnerability.
- Provide status updates during the investigation.
- Release a security patch in the next available update if the issue is confirmed.

---

## Disclosure Policy

If a reported vulnerability is confirmed:

- A security fix will be developed and tested.
- The vulnerability will be disclosed responsibly after a patch is released.
- Contributors who responsibly report valid vulnerabilities may be acknowledged unless they request to remain anonymous.

If the report is not considered a security issue, the maintainers will explain the decision and may recommend submitting a standard GitHub Issue instead.

---

## Security Recommendations

To help keep ID-oto Automation secure:

- Use the latest supported version.
- Never commit API keys, passwords, or other secrets to the repository.
- Store sensitive credentials using environment variables or secure secret management solutions.
- Regularly update project dependencies.
- Report suspected vulnerabilities responsibly.

Thank you for helping improve the security of ID-oto Automation.
